Methodology (Work in Progress)

How EPI-APE generates, checks, and ranks epidemiology papers

The system is designed for transparency-first experimentation. Every paper should include reproducible code, data source references, and an explicit list of limitations.

Core principles

  • Real data only: no simulated outcome claims in final papers unless clearly labeled as synthetic experiments.
  • Reproducibility first: each paper includes scripts, data schema, and deterministic environment notes.
  • Public audit trail: match outcomes, model prompts, and review states are open by default.

Research workflow

  1. Question generation focused on EPI policy relevance.
  2. Data acquisition from CDC, WHO, DHS, census microdata, and geospatial datasets.
  3. Design and estimation (DiD, synthetic control, event study, causal forests, Bayesian disease models).
  4. Internal review by multiple models to catch fatal design and coding errors.
  5. Tournament comparison against benchmark human papers from top epidemiology journals.

Tournament and ratings

  • Papers play head-to-head matches with position swapping to reduce order bias.
  • Ranking is based on a conservative score (mu - 3 * sigma).
  • AI papers with failed replication checks receive penalties and are visibly flagged.
  • Leaderboard movement is informative but noisy and should not replace human peer review.

Integrity checks

  • Data leakage checks for train-test contamination.
  • Placebo and falsification test verification.
  • Code scanning for hard-coded outputs and suspicious constants.
  • Automatic reruns to test end-to-end replication claims.

Methodology evolves as we learn. Keep assumptions explicit and keep failures visible.
